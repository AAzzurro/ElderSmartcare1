# "问大夫"功能 - 用药信息关联实现

## 功能概述
在老人端和护工端的"问大夫"页面提问时，系统现在会自动关联并检索使用者的用药信息（说明书中的内容），使得 chatbot 在回答问题时能够考虑使用者的用药情况，并在回答中清晰地体现它已经搜索了使用者的用药信息。

## 实现原理

### 核心组件变更

#### 1. **ai_agents.py** - 新增两个功能函数

##### a. `get_user_medicines_context(bed_no="", resident_name="")`
- **功能**：根据患者身份（床号+姓名）检索 MEDICINE_DB 中的用药信息
- **参数**：
  - `bed_no=""`：患者床号（可选，护工端使用）
  - `resident_name=""`：患者姓名（可选，护工端使用）
- **返回**：格式化的用药信息文本，包含药品名称、用法、禁忌、服药时间等
- **逻辑**：
  - 当两个参数都为空时，返回所有用药记录（老人端场景）
  - 当传入床号或姓名时，返回匹配的用药记录（护工端场景）
  - 若无匹配记录，返回空字符串

##### b. `chat_with_rag()` 函数升级
- **新增参数**：
  - `bed_no=""`：患者床号
  - `resident_name=""`：患者姓名
- **功能改进**：
  1. 检索用户用药信息并计算用药记录数
  2. 在回答开头显示"🔍 已检索 X 条用药记录"的摘要
  3. 将完整的用药信息加入到系统提示词中
  4. 强制医生在回答时考虑患者的实际用药情况
- **系统提示词更新**：
  ```
  你已经搜索了患者当前的用药信息，请在回答中：
  1. 首先告知患者你已经查看了他们的用药记录
  2. 根据他们的实际用药情况给出专业建议
  3. 特别关注药物相互作用和禁忌事项
  ```

#### 2. **app.py** - 界面和事件绑定升级

##### a. 核心包装函数

```python
def chat_elder_wrapper(message, history):
    """老人端包装函数：查询所有用药信息（不限制床号和姓名）"""
    yield from chat_with_rag(message, history, bed_no="", resident_name="")

def chat_nurse_wrapper(message, history, bed_no, resident_name):
    """护工端包装函数：根据指定患者查询用药信息"""
    bed_no = str(bed_no or "").strip()
    resident_name = str(resident_name or "").strip()
    yield from chat_with_rag(message, history, bed_no=bed_no, resident_name=resident_name)
```

##### b. 老人端"问大夫"界面增强
```
- 新增说明："您的用药记录将自动被检索，医生会根据您的实际情况给出专业建议。"
- 自动查询所有该用户的用药信息
- 不需要手动输入任何患者信息
```

##### c. 护工端"问大夫"界面增强
```
- 新增患者信息输入框：
  * 🛏️ 患者床号（可选）
  * 👴 患者姓名（可选）
- 新增说明："请输入患者信息，系统将自动查询其用药记录并帮助医生制定个性化建议。"
- 护工可以输入床号和姓名以查询特定患者的用药信息
```

## 用户流程

### 老人端用户流程
1. 老人进入"问大夫" tab
2. 输入问题（如："我胃疼能吃布洛芬吗？"）
3. 系统自动检索老人的所有用药记录
4. 医生 AI 看到检索结果（显示"🔍 已检索 X 条用药记录"）
5. 医生 AI 基于老人的实际用药情况回答问题

### 护工端用户流程
1. 护工进入"问大夫（护工）" tab
2. 输入患者床号（如："301床"）和患者姓名（如："王大爷"）
3. 输入问题（如："患者在肝功能不好的情况下能吃这种药吗？"）
4. 系统自动检索该患者的用药记录
5. 医生 AI 看到检索结果（显示"🔍 已检索 X 条用药记录"）
6. 医生 AI 基于患者的实际用药情况回答问题

## 技术细节

### 用药信息检索逻辑
- **老人端**：`bed_no=""` 和 `resident_name=""`，检索所有用药记录
- **护工端**：根据输入的床号和姓名精准匹配，只检索该患者的用药记录
- **匹配规则**：
  - 如果只提供床号，则检索该床号的所有药物
  - 如果只提供姓名，则检索该姓名的所有药物
  - 如果同时提供，则同时匹配（AND 逻辑）

### 系统提示词架构
系统提示词包含三部分：
1. **基础指令**：医生身份、语言风格、客户群体特征
2. **用药信息**：患者的实际用药记录
3. **本地知识库**：高危词汇警告（如布洛芬、阿莫西林等）

这个三层架构确保医生 AI 在考虑综合信息的基础上给出专业建议。

### 流式回复处理
- 医生回答开始时显示："🔍 已检索 X 条用药记录"
- 然后实时流式输出医生的建议
- 如果无用药记录，显示："🔍 未检索到用药记录"

我建议：
1. 先服用温和的胃药或喝点温水
2. 尽快联系您的主治医生或社区卫生员

## 安全性和合规性

### 数据保护
- 用药信息仅在同一会话内使用
- 不会记录用户提出的具体问题
- 符合医疗隐私要求

### 医疗责任声明
- 系统明确指示用户联系真实医生或医疗机构以获取专业医疗建议
- AI 回答仅供参考，不能替代医生诊治
- 紧急情况下明确建议就医
